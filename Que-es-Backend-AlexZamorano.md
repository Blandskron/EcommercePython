# Backend Developer - Mi Primer proyecto

## 🧠 ¿Qué hace un Desarrollador Backend?

El motor invisible del software.
Un **desarrollador backend** es el "mecánico" o el motor detrás de una aplicación. 
Mientras el usuario interactúa con la interfaz (frontend), el backend se encarga de procesar datos, gestionar la seguridad y conectar todo con la base de datos.

Un desarrollador **backend** construye la parte interna de una aplicación.  
Es la parte que el usuario no ve, pero que hace que todo funcione.
Sin backend, una app tendría solo la parte visual, pero no podría funcionar de verdad.

Imagina un videojuego en línea:

Tu personaje camina y salta en la pantalla (frontend), pero atrás están las reglas del juego (puntos, vidas, niveles) y la comunicación con otros jugadores.
 Esa parte es el backend: comprueba si tienes suficientes puntos para saltar, guarda el puntaje en la base de datos, y envía la información al servidor del juego. Tú solo ves el juego, pero el backend es el que hace las cuentas en secreto.


>### 🎮 Ejemplo: ¿Qué pasa cuando saltas en un juego?
>
>Para entender el backend, imagina que presionas el botón de "Saltar":
>
>- 🕹️ **Frontend:** Envía "el jugador quiere saltar"
>- ⚙️ **Backend:** Valida reglas:
>     - ¿Tiene energía?
>     - ¿Está en el suelo?
>     - ¿Hay obstáculo?
>- 🗄️ **Base de datos:** Guarda cambios
>- ✅ **Resultado:** Se ejecuta el salto

## 🛠️ Funciones Principales

En el día a día (especialmente en entornos **Scrum**), mis tareas incluyen:

* **APIs REST:** Construir los puentes de comunicación.
* **Bases de Datos:** Diseñar dónde y cómo se guarda la información.
* **Lógica de Negocio:** Programar las reglas (ej: calcular un descuento o validar un RUT).
* **Seguridad:** Controlar quién entra y qué puede ver.

## 🤝 Colaboración en el Equipo

En un equipo profesional, el backend no trabaja solo; es el puente entre distintas áreas:

#### 💼 Negocio y Gestión

* **Product Owner:** Traduzco sus requerimientos comerciales en lógica de programación.
* **Scrum Master:** Colaboro para identificar bloqueos técnicos y mantener el ritmo de entrega.

#### 🎨 Diseño y Experiencia (UX/UI)

* **Diseñadores:** Aseguro que el servidor entregue los datos necesarios para que la interfaz sea rápida y fiel al diseño.
* **Frontend:** Acordamos el "contrato" de la API (qué datos viajan y cómo) para que la integración sea perfecta.

#### 🛡️ Calidad y Estabilidad

* **Líder Técnico:** Sigo sus lineamientos de arquitectura para escribir código limpio y escalable.
* **QA (Testers):** Entrego ambientes de prueba y soluciono rápidamente cualquier error (*bug*) reportado.

#### 🚀 Infraestructura (DevOps)

* **DevOps Engineer:** Trabajo con contenedores (**Docker**) y pipelines de **CI/CD** para que el código llegue a producción de forma automática y segura.
